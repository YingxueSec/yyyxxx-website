# Yingxue AiSQL - AI 驱动的 SQL 注入自动化检测与利用工具

## 项目简介

Yingxue AiSQL 是一款基于人工智能的 SQL 注入自动化检测与利用工具，集成了传统漏洞检测技术与 AI 驱动的智能利用能力，为安全测试人员提供高效、智能的 SQL 注入漏洞发现与利用解决方案。

## 核心特性

### 🤖 AI 驱动的智能利用
- 基于 SiliconFlow GLM-4.6 大语言模型
- 自动化迭代构造 payload
- 智能分析响应并动态调整策略
- 自动提取数据库版本、用户名、数据库名等敏感信息

### 🎯 全流程自动化
- **观察阶段**：自动捕获 HTTP 流量
- **探测阶段**：快速检测可疑注入点
- **发现阶段**：确认漏洞并生成报告
- **利用阶段**：AI 驱动的自动化利用
- **一键推进**：Pipeline 视图支持流程化管理

### 🔍 多种注入类型支持
- **错误回显注入**：基于 `updatexml`/`extractvalue` 等函数
- **布尔盲注**：长度差异、状态码差异、模式匹配
- **时间盲注**：延迟判定（`sleep` 函数）
- **联合注入**：AI 驱动的列数探测与数据提取

### 🌐 现代化 Web 界面
- 实时流量监控（Exchanges）
- 漏洞发现列表（Findings）
- 流程化视图（Pipeline）
- 手动/自动利用工具（Exploit）
- 审计日志（Audits）
- 灵活配置（Settings）

### 🔌 无缝集成 Burp Suite
- 提供 Burp Suite 扩展
- 自动转发代理历史
- 无需修改现有工作流
- 支持被动流量分析

### 📦 开箱即用
- 单文件部署，前后端合一
- 跨平台支持（macOS / Linux / Windows）
- 内置测试 API Key（100 元额度）
- 零配置启动

## 技术架构

### 后端技术栈
- **语言**：Go 1.23
- **Web 框架**：标准库 `net/http`
- **数据存储**：SQLite（modernc.org/sqlite 纯 Go 实现）
- **AI 集成**：SiliconFlow API
- **并发控制**：令牌桶 + 信号量

### 前端技术栈
- **框架**：React 18
- **构建工具**：Vite 5
- **UI 组件**：Radix UI
- **样式**：Tailwind CSS
- **图标**：Lucide React

### 核心模块
- **detect**：检测引擎，支持多种注入类型
- **exploit**：利用引擎，AI 驱动的迭代利用
- **control**：执行控制，速率限制与并发管理
- **ai**：AI 客户端，与大语言模型交互
- **store**：数据存储，内存 + SQLite 持久化
- **whitelist**：白名单管理，动态更新
- **web**：Web 服务，SSE 实时推送

## 工作流程

```
1. Burp Suite 捕获流量
   ↓
2. 自动转发到 Yingxue AiSQL
   ↓
3. 第一层快速检测（布尔/时间盲注）
   ↓
4. 发现可疑注入点 → 生成 Finding
   ↓
5. Pipeline 一键推进到 AI 利用
   ↓
6. AI 自动化构造 payload 并提取数据
   ↓
7. 结果自动提升为 Finding
```

## 快速开始

### 1. 下载运行
```bash
# macOS (M1/M2/M3)
./yingxue-aisql-darwin-arm64

# Linux
./yingxue-aisql-linux-amd64

# Windows
yingxue-aisql-windows-amd64.exe
```

### 2. 安装 Burp 扩展
在 Burp Suite 中加载 `YingxueForwarder.jar`

### 3. 访问 Web 界面
打开浏览器访问 `http://127.0.0.1:19081`

### 4. 开始测试
通过 Burp 代理访问目标网站，在 Web 界面查看检测结果

## 配置说明

### 基础配置
```env
PORT=19081                    # 服务端口
ENABLE_AI=true                # 启用 AI 功能
SILICONFLOW_API_KEY=sk-xxx    # API Key（已内置测试 Key）
```

### 高级配置
- **len_diff_threshold**：布尔盲注长度差异阈值（默认 30）
- **delay_ms_threshold**：时间盲注延迟阈值（默认 1200ms）
- **expect_pattern**：预期响应特征（用于模式匹配）
- **白名单规则**：限制测试范围
- **速率限制**：控制请求频率
- **并发控制**：限制同时进行的测试数量
